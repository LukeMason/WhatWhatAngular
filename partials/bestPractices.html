Best Practices

  • Namespace distributed code. The ng- is reserved for core directives, pick a namespace that makes sense for you're project and scope.

  • Extend directives by using Directive Controllers.

  • Leverage modules. Break you code into related modules so you can add and remove functionality without breaking the whole app.

Do this Not that

  • Don't use the global function version of controllers. By default angular won't complain but when you minify your code everything will stop working.

  • Don't do $(this). If you are needing a jQuery wrapped version of the element you're probably doing something wrong. (see the jquery page for more)
